Changelog

v1.0.1 March 29, 2026

Added

  • Dynamic Remote Rules Syncing: The extension now utilizes background alarms to silently download the newest ad-blocking and cosmetic network rules directly from a secure API, letting us deploy rapid filter updates without requiring waiting for an extension update review.
  • Added staged retry handling for the initial remote rules download. Fresh installs now retry every minute for the first 10 failed attempts, then every 5 minutes until the first synced ruleset is available.
  • Added a "Block notification prompts" toggle to prevent sites from requesting invasive browser notification permissions. When enabled, it spoofs a "denied" state directly to any site attempting to prompt the user.
  • Added tracking of daily blocked ads and cookie overlays to local storage. It keeps a rolling history of the last 400 days.
  • Added "declarativeNetRequestFeedback" permission to intercept and count network-level ad blocks accurately.
  • Added a small "Enable" button in the popup when blocking is globally disabled, allowing users to re-enable directly.
  • Added "Block tracking" toggle (enabled by default) with a separate static ruleset for analytics and tracking scripts (Google Analytics, Google Tag Manager, Microsoft Clarity, comScore, Bing telemetry).
  • Custom selectors, custom block domains, and custom video ad domain sections on the settings page are now collapsed by default and expand with a smooth slide-down animation when their heading is clicked.
  • Added a subtle "rules last updated" timestamp and refresh button at the bottom of the settings page, allowing users to see when rules were last synced and manually trigger a refresh.

Fixed

  • Fixed Windows packaged-build dependency setup by caching the build-only npm tools outside the synced workspace so `build_min.js` no longer fails when OneDrive or another process locks `node_modules`.
  • Fixed a bug where allowlisted (disabled) sites would infinitely increment the "Ads removed" count by mischaracterizing regular background network requests as ad blocks.
  • Fixed inaccurately low "Ads removed" counter caused by queries occasionally failing or elements being dynamically removed from the DOM before being counted.
  • Updated the "Ads removed" counter to factor in both cosmetically hidden ads on the page and ads blocked natively at the network level.
  • Fixed "No tab with id" errors when interacting with closed tabs in the popup and service worker.
  • Fixed the "Block notification prompts" toggle so it now persists correctly through the shared settings pipeline and applies consistently across the options page, popup state, and content script.
  • Fixed notification prompt blocking so it can be removed again on the current page when the user disables the setting or allowlists the site.
  • Fixed cosmetic and cookie-consent rule refresh behavior so synced remote selector updates apply to already-open tabs without requiring a manual page reload.
  • Fixed wildcard allowlist entries so network-level allow rules now match the same subdomain-only scope used by the UI and content script state.
  • Fixed notification permission restoration so disabling the feature removes the page-level override cleanly instead of leaving stale permission state behind.
  • Fixed popup and options error paths so temporary message failures do not leave controls stuck in stale visual state or permanently block saving with default values.
  • Fixed custom block-domain and video-ad domain persistence so only valid DNR filter patterns are stored and replayed back into dynamic rules.
  • Fixed remote rule ingestion so malformed or empty payloads no longer overwrite the last known good ruleset.
  • Fixed dynamic rule generation to cap remote, custom, and video rule groups within their reserved ID ranges before syncing them into DNR.
  • Fixed partial remote payload handling so selectors, cookie rules, and video-domain lists are preserved when the API only returns some sections.
  • Fixed first-run sync recovery so brand new installs keep retrying automatically instead of sitting idle for hours after an initial network failure.
  • Fixed packaged-build popup counts by reading matched DNR rules for the active page instead of depending only on the unpacked debug event.
  • Fixed cosmetic and cookie marker cleanup so selector removals and attribute-only DOM changes do not leave stale blocked markers behind.
  • Fixed production fallback behavior so missing synced rules no longer spam failed requests for stripped local JSON files.
  • Fixed a content-script observer regression that could rescan ad-heavy pages in a tight loop and push Chrome into the “force close this tab” path.
  • Fixed popup ad counts on heavy pages by reading the current blocked-marker state from the page instead of trusting only the in-memory tracker set.

Changed

  • Updated the settings page to refresh the "rules updated" timestamp immediately when a background rules sync completes.
  • Updated the popup and settings page to show when the initial rules download is still pending and when the next retry will run.
