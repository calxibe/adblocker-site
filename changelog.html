Changelog

v1.0.1 March 29, 2026

Added

  • Dynamic Remote Rules Syncing: The extension now silently downloads the newest ad-blocking and cosmetic rules from a secure API, letting us deploy filter updates without requiring an extension update.
  • Added a "Block notification prompts" toggle to prevent sites from requesting invasive browser notification permissions.
  • Added tracking of daily blocked ads and cookie overlays to local storage, keeping a rolling history of the last 400 days.
  • Added a small "Enable" button in the popup when blocking is globally disabled, allowing users to re-enable directly.
  • Added "Block tracking" toggle (enabled by default) with a separate ruleset for analytics and tracking scripts (Google Analytics, Google Tag Manager, Microsoft Clarity, comScore, Bing telemetry).
  • Custom selectors, custom block domains, and custom video ad domain sections on the settings page are now collapsed by default and expand with a smooth animation.
  • Added a "rules last updated" timestamp and refresh button at the bottom of the settings page.

Fixed

  • Fixed a bug where allowlisted sites would infinitely increment the "Ads removed" count.
  • Fixed inaccurately low "Ads removed" counter and updated it to factor in both cosmetically hidden ads and network-level blocks.
  • Fixed the "Block notification prompts" toggle so it persists correctly and can be cleanly disabled per-site.
  • Fixed cosmetic and cookie-consent rule refresh so synced selector updates apply to already-open tabs without a page reload.
  • Fixed wildcard allowlist entries so network-level allow rules match the same scope used by the UI.
  • Fixed a content-script observer regression that could rescan ad-heavy pages in a tight loop and spike CPU usage.

Changed

  • Updated the website with a direct Chrome Web Store install link now that the extension has been approved on Chrome.
  • Updated the settings page and popup to show rule sync status, last-updated timestamps, and retry timing.
  • Clarified the privacy policy to document remote rule downloads, browser-local storage usage, and the no-IP-log policy for the rules API.
v1.0.0 March 25, 2026
  • Initial Manifest V3 extension with static network blocking rules.
  • Cosmetic filtering with built-in site selectors and support for custom selectors.
  • Cookie consent banner hiding and page unlock handling.
  • Popup for per-site allowlisting and blocked item status.
  • Options page for global settings, allowlisted domains, custom block domains, and custom video ad domains.
  • First-install onboarding that opens the settings page with instructions for pinning the extension icon.
